hexo文章快速生成指令查阅

先配置环境

  • github端：
  1. 新建一个私有仓库放源码，防止你的内容被篡改，名 hexo-source（随意取） ，不要readme
  2. 新建一个公有的仓库（GitHub Pages要求公开才能访问），名为 用户名.github.io ，这里用户名，是要和github上的用户名对应的，就是登录github的用户名。这个仓库作用是用自动化脚本从源码位置搞成网页放过去，不要readme
  3. 生成令牌
    Github头像→Settings → Developer settings → Personal access tokens → Tokens(classic) → 勾选 repo全部权限 → 复制令牌
  4. 把Token密钥存进私有仓库
    私有仓库 → Settings → Secrets and variables → New repository secret → Name 填 PAGES_TOKEN → Value 贴刚才的令牌
  • 本地：
  1. 装Node.js
    https://nodejs.org/
  2. 装git
  3. 装hexo
  • 自动化部署脚本（GitHub Actions 自动部署）
    每次你推送源码到私有仓库 ，GitHub Actions 会自动构建静态文件，并推送到公开仓库。别人只能看到构建后的静态文件，看不到源码和文章原始内容。

如果是用Windows PowerShell

会出现禁止运行脚本的报错
用下面代码临时允许脚本
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
然后继续操作即可
也可以永久运行
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
