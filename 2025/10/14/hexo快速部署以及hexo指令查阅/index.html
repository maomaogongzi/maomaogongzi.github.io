先配置环境

  • github端：
  1. 新建一个私有仓库放源码，防止你的内容被篡改，名 hexo-source（随意取） ，不要readme
  2. 新建一个公有的仓库（GitHub Pages要求公开才能访问），名为 用户名.github.io ，这里用户名，是要和github上的用户名对应的，就是登录github的用户名。这个仓库作用是用自动化脚本从源码位置搞成网页放过去，不要readme
  3. 生成令牌
    Github头像→Settings → Developer settings → Personal access tokens → Tokens(classic) → 勾选 repo全部权限 → 复制令牌
  4. 把Token密钥存进私有仓库
    私有仓库 → Settings → Secrets and variables → New repository secret → Name 填 PAGES_TOKEN → Value 贴刚才的令牌
  • 本地：
  1. 装Node.js
    https://nodejs.org/
  2. 装git
  3. 装hexo
  • 自动化部署脚本（GitHub Actions 自动部署）
    每次你推送源码到私有仓库 ，GitHub Actions 会自动构建静态文件，并推送到公开仓库。别人只能看到构建后的静态文件，看不到源码和文章原始内容。

如果是用Windows PowerShell

会出现禁止运行脚本的报错
用下面代码临时允许脚本
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
然后继续操作即可
也可以永久运行
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

初始化git的问题

初始化git的时候，应该是在里面那层文件夹里初始化，源文件所在的地方，不要到外面那一层自己新建文件夹初始化，理论上也可以，但是脚本里的位置是以里层做的，所以如果git到外一层，可能会因为路径不对，脚本报错。注意这个问题

主题设置

  • Themes | Hexo
    一般从官方主题那就有很多，也可以找找很多没有收录进来的主题。
  • 可以看看github上的主题说明，以下是比较通用的部分。

以Meow主题为例

  1. 首先安装git该主题的库，指定下载位置，不然下载下来是hexo-theme-meow，和配置文件会对应不上
    git clone -b main https://github.com/chanwj/hexo-theme-meow.git themes/meow
  2. 修改根目录下的的配置文件_config.yml，将主题更换为meow
  3. 然后进行渲染npm install hexo-renderer-pug --save
  4. 如果没有Stylus渲染器（在package.json中检查）
    npm install hexo-renderer-stylus --save
  5. 以上只是安装步骤，但是安装完还需要配置，现在还不能用
  6. 由于clone主题仓库的时候带了一个.git的隐藏文件夹，刚练手建议先删了，不然会有警告。如果需要更新这个仓库，再说。
  7. 第一次使用会报错，要先去主题配置文件下找到sidebar的links的list，在该条目下面不能为空，要写一些东西，写一些个人的仓库地址，知乎地址等内容。总之，不允许为空
  1. 另一个报错是搜索插件问题，先npm install hexo-generator-searchdb --save然后需要去根目录的配置文件下面，添加下面几行,注意，yml文件有严格格式要求，缩进要注意，不然依然会报错。（第7点同样也是要注意格式）

回到主题的配置文件，要把search段enable改成true

  1. 然后就是清理缓存，重新生成静态文件，启动服务，最后本地访问看看。
  2. 最后推送到github上时候，会遇到一些问题，如果遇到你的主题文件夹在github上打不开，是灰色，或者出现了个箭头，但是看不到子文件夹，是因为这个主题是从别人的仓库里clone下来的，会带上.git的隐藏文件夹，会被github认为是嵌套仓库的机制。主仓库只会记录子模块的仓库地址和版本指针，而非子模块的文件内容。所以在本地文件的主题文件夹里的.git文件删掉。但是此时你的本地仓库不认为这是一次改动，所以推不上去，他无法识别到变化。这是因为有git的子模块混存，所以对缓存进行删除。git rm --cached themes/meow
